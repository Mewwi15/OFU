1. ขอบเขตและวิธีทดสอบ

ทดสอบเสมือนผู้ใช้จริงทุกขั้นตอน โดยสร้างบัญชีลูกค้าทดสอบ + แอดมินทดสอบ + สินค้าทดสอบแยกจากข้อมูลจริง และลบทิ้งทั้งหมดหลังจบการทดสอบ (ออเดอร์/บิล/สต็อก/เลขรันนิ่งถูกคืนค่าเดิมครบ ตรวจยืนยันแล้ว)

ด้านเคสที่ทดสอบผล
แอป — สมัคร/เข้าสู่ระบบ/PINสมัครสมาชิก, รหัสผิด, เข้าสู่ระบบ, ตั้ง PIN + ยืนยัน, PIN ผิด/ถูก, ล็อกเมื่อพักแอป > 2 นาที, ล็อกตอนเปิดแอปใหม่, ขอสิทธิ์แจ้งเตือนผ่าน
แอป — เลือกซื้อจนจบออเดอร์เปิดดูสินค้า, ค้นหา, เพิ่มลงตะกร้า, ปรับจำนวน, ขั้นต่ำสั่งซื้อ, แถบส่งฟรี, สรุปยอด, ปักหมุดที่อยู่บนแผนที่ + GPS + ที่อยู่อัตโนมัติ, สั่งแบบเก็บเงินปลายทาง, สั่งแบบพร้อมเพย์ + แนบสลิป, ติดตามออเดอร์, ยกเลิกออเดอร์ (สต็อกคืนอัตโนมัติ), ให้คะแนนหลังส่งสำเร็จ, ตะกร้าคงอยู่หลังปิดแอปผ่าน
เว็บ — ขายหน้าร้าน (POS)ยิงบาร์โค้ดผ่านช่องค้นหา, บิลหลายรายการ, กันชำระเมื่อรับเงินไม่พอ, เงินทอน, ใบเสร็จ + บาร์โค้ดบิล, ขายต่อ, บิลขายย้อนหลัง, คืนเงิน (สต็อกคืนอัตโนมัติ)ผ่าน
เว็บ — ออเดอร์ออนไลน์ตรวจสลิป (อนุมัติ), เลื่อนสถานะครบวงจร ยืนยัน → จัดเตรียม → ไรเดอร์ → นำส่ง → สำเร็จ, แก้ไขสินค้า/ราคา, ทุกหน้าเปิดได้ไม่มี error (สินค้า หมวดหมู่ รายงาน ตั้งค่า แบนเนอร์ ประกาศ เครดิตร้าน scan-lab)ผ่าน
ระบบเชื่อมถึงกันอนุมัติ/เลื่อนสถานะบนเว็บ → หน้าติดตามในแอปเปลี่ยนเองทันทีไม่ต้องรีเฟรช, แจ้งเตือน push เด้งเข้ามือถือ (ไอคอนเสือ), สต็อกร่วมหน้าร้าน + ออนไลน์ตัดจากก้อนเดียวกัน, แก้ราคาบนเว็บเห็นในแอปทันทีผ่าน

2. บั๊กที่พบและแก้เสร็จแล้ว

ระดับอาการการแก้สถานะ
วิกฤต POS บนเว็บจริงกดชำระเงินแล้ว error ทุกบิล — migration ก่อนหน้า (0038) เผลอสร้างฟังก์ชันขายซ้ำ 2 เวอร์ชันในฐานข้อมูล ระบบเลือกไม่ได้ และเวอร์ชันที่งอกมายังทำระบบจ่ายแบบผสม (split payment) หายด้วย migration 0041 — ลบเวอร์ชันเก่า รวมการแก้ต้นทุนทศนิยมเข้าเวอร์ชันที่ถูกต้อง แก้แล้ว + ขึ้นระบบจริง
วิกฤต แอปเด้ง (crash) เมื่อเปิดหน้าปักหมุดที่อยู่โดยยังไม่ให้สิทธิ์ตำแหน่ง — ลูกค้าติดตั้งใหม่ที่เพิ่มที่อยู่ครั้งแรกเจอทุกคน เปิดจุดตำแหน่งบนแผนที่เฉพาะเมื่อได้รับสิทธิ์แล้ว แก้แล้ว
กลาง ตาราง "สินค้าขายดี" ในหน้ารายงานนับเฉพาะบิลหน้าร้าน ทั้งที่หัวข้อบอกว่ารวมออนไลน์ — ยอดในหน้าเดียวกันขัดกันเอง migration 0042 — รวมออเดอร์ออนไลน์ที่ชำระแล้วเข้าอันดับ แก้แล้ว + ขึ้นระบบจริง
กลาง การ์ดออเดอร์ในแอปนับจำนวนชิ้นเป็นจำนวนรายการ — สั่งสินค้า 1 อย่าง 5 ชิ้น ขึ้นว่า "และอีก 4 รายการ" แยกตัวนับ "รายการสินค้า" ออกจาก "จำนวนชิ้น" แก้แล้ว
เล็ก กรอกรหัสผ่านผิดตอนเข้าสู่ระบบ แต่ข้อความ error ขึ้นเป็นเงื่อนไขรหัสผ่าน ("อย่างน้อย 6 ตัวอักษร") ทำให้เข้าใจผิด แก้เป็น "อีเมลหรือรหัสผ่านไม่ถูกต้อง" แก้แล้ว
เล็ก มีอิโมจิหลุดในหน้ารายงานของเว็บแอดมิน 1 จุด (ขัดกับแนวทางแบรนด์) ลบออก แก้แล้ว (รอ deploy)

ปรับปรุง UX เพิ่มเติมระหว่างทดสอบ

  • สั่งซื้อโดยยังไม่มีที่อยู่ — เดิมแจ้งเตือนเฉย ๆ ตอนนี้มีปุ่ม "เพิ่มที่อยู่" พาไปหน้าปักหมุดทันที

3. หลักฐานการทดสอบ

4. ข้อสังเกตที่ควรตัดสินใจก่อนเปิดใช้จริง

บัญชีพร้อมเพย์ยังเป็นตัวอย่าง — หน้า QR ในแอปแสดง "อู้ฟู่ จำกัด · 081-234-5678" ซึ่งเป็นค่าตัวอย่าง ต้องตั้งบัญชีรับเงินจริงในหน้าตั้งค่าก่อนรับออเดอร์ออนไลน์จริง
ข้อมูลไรเดอร์เป็นค่าจำลอง — ชื่อ/รูปไรเดอร์ ("พีรพล จันทร์ดี") และตำแหน่งบนแผนที่เป็นข้อมูลตัวอย่างไปก่อน จนกว่าระบบไรเดอร์จริงจะเสร็จ ควรแจ้งให้เจ้าของทราบตอนเดโม
คะแนนรีวิวยังไม่เก็บเข้าฐานข้อมูล — ดาว + คอมเมนต์หลังส่งสำเร็จเก็บไว้ในเครื่องลูกค้าเท่านั้น ควรต่อเข้าหลังบ้านในเฟสถัดไป
โหมดส่งพัสดุ (ออนไลน์ต่างจังหวัด) ยังทดสอบไม่ได้ — รอ Flash Express merchant credentials จากเจ้าของร้าน
จุดขัดเกลาเล็กน้อย — หน้ายืนยันยกเลิกออเดอร์ค่อนข้างโล่ง, ใบเสร็จย้อนหลังไม่แสดงรับเงิน/เงินทอน (แสดงเฉพาะยอดชำระ), ปุ่ม "อ่านเพิ่มเติม" แสดงแม้สินค้าไม่มีคำอธิบาย

5. การเก็บกวาดหลังทดสอบ

ข้อมูลทดสอบทั้งหมดถูกลบออกจากระบบจริงและตรวจยืนยันแล้ว: บัญชีทดสอบ 2 บัญชี, สินค้าทดสอบ 2 รายการ, ออเดอร์ทดสอบ 2 ออเดอร์, บิล POS ทดสอบ 1 บิล, ไฟล์สลิป, ประวัติการทำงานของแอดมินทดสอบ — สต็อกสินค้าจริงคืนค่าเดิมครบ และเลขรันออเดอร์/บิลถูกรีเซ็ตกลับ (ออเดอร์ถัดไปเริ่ม OF00001 ตามเดิม)
