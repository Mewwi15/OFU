## 2. Data Safety form (Google Play) — คำตอบตามที่แอปทำจริง

**สรุปหลัก:**
- เก็บข้อมูล: **ใช่**
- แชร์ข้อมูลกับบุคคลที่สาม: **ใช่** (เฉพาะขนส่ง + ผู้ให้บริการระบบ — ไม่ใช่เพื่อโฆษณา)
- เข้ารหัสระหว่างส่ง (in transit): **ใช่**
- ผู้ใช้ขอลบข้อมูลได้: **ใช่** (ลบบัญชีในแอป)
- ไม่ขายข้อมูล · ไม่มีโฆษณา

**ประเภทข้อมูลที่เก็บ (collected / shared / purpose):**

| ประเภท | เก็บ | แชร์ | วัตถุประสงค์ |
|---|---|---|---|
| ชื่อ | ✅ | ✅ ขนส่ง | สั่ง/ส่งสินค้า (App functionality) |
| อีเมล | ✅ | ✕ | จัดการบัญชี |
| เบอร์โทร | ✅ | ✅ ขนส่ง | ติดต่อจัดส่ง |
| ที่อยู่ | ✅ | ✅ ขนส่ง | จัดส่งสินค้า |
| ตำแหน่ง (คร่าว + แม่นยำ) | ✅ | ✕ | ปักหมุดที่อยู่จัดส่ง (App functionality) — ไม่ใช่โฆษณา |
| รูปภาพ (สลิป + โปรไฟล์) | ✅ | ✕ | ยืนยันการชำระเงิน / โปรไฟล์ |
| ประวัติการสั่งซื้อ | ✅ | ✕ | ให้บริการคำสั่งซื้อ |
| ข้อความในแอป (แชต) | ✅ | ✕ | สนับสนุนลูกค้า |
| Device ID / push token | ✅ | ✕ | ส่งการแจ้งเตือน |

*หมายเหตุ:* ตำแหน่ง = "App functionality" (ปักหมุดที่อยู่) ห้ามติ๊ก Advertising
การแชร์กับขนส่ง/Supabase/Google/Apple = "service provider" ไม่ใช่ "sold"
