# 02 — Personas: อู้ฟู่ (Oofoo)

> ใช้ persona เป็นตัวขับ requirements + acceptance criteria ทุกข้อควรตอบโจทย์ persona อย่างน้อยหนึ่งคน

## P1 — "น้องแนน" · ลูกค้าหลัก (วัยทำงาน 28 ปี)
- **บริบท:** ทำงานออฟฟิศ กลับบ้านดึก ไม่อยากแวะร้าน อยากสั่งของกินของใช้ส่งถึงคอนโด
- **เป้าหมาย:** สั่งเร็ว จ่าย PromptPay จบใน 2 นาที รู้ว่าของจะถึงเมื่อไหร่
- **Pain:** แอปช้า/ขั้นตอนเยอะ, ไม่รู้สถานะออเดอร์, ค่าส่งไม่ชัด
- **ขับ requirement:** checkout สั้น, โหมดเดลิเวอรี่ + ค่าส่ง/ส่งฟรีชัดเจน, ติดตามสถานะ + push

## P2 — "ป้าสมศรี" · ลูกค้าสูงวัย (58 ปี)
- **บริบท:** อยู่ละแวกร้าน ชอบสั่งออนไลน์แล้วเดินไปรับเอง สายตาเริ่มไม่ดี
- **เป้าหมาย:** ตัวหนังสือใหญ่อ่านง่าย ปุ่มกดง่าย จ่ายด้วยการโอน+แนบสลิป
- **Pain:** ตัวอักษรเล็ก, ปุ่มเล็กกดพลาด, สีอ่อนมองไม่เห็น
- **ขับ requirement:** **รองรับฟอนต์ใหญ่ (dynamic type), touch target ≥ 44pt, contrast ผ่าน WCAG AA**, โหมดออนไลน์รับที่ร้าน + PromptPay/สลิป
- *(เชื่อมโยงตรงกับ a11y findings ในรายงานรีวิว — persona นี้คือเหตุผลทางธุรกิจของการแก้ a11y)*

## P3 — "เฮียอู้ฟู่" · เจ้าของร้าน / แอดมิน (45 ปี) ✅ *(อยู่ในขอบเขต v1 — เว็บแอดมิน)*
- **บริบท:** ดูแลร้านเอง รับออเดอร์ จัดของ อัปเดตสต็อก/ราคา ตรวจสลิป
- **เป้าหมาย:** เห็นออเดอร์เข้าทันที, อัปเดตสถานะ (รับ→เตรียม→ส่ง/พร้อมรับ), จัดการสินค้า+ราคา, ยืนยันการชำระเงิน
- **Pain:** ตรวจสลิปด้วยมือเยอะ, ของหมดแต่ลูกค้ายังสั่งได้, แก้ราคาหลายที่
- **ขับ requirement:** ระบบจัดการแคตตาล็อก/ออเดอร์, แจ้งเตือนออเดอร์ใหม่, เวิร์กโฟลว์ตรวจชำระเงิน

## P4 — "ไรเดอร์" · ผู้ส่งสินค้า ✅ *(อยู่ในขอบเขต v1 — แอปไรเดอร์ เดลิเวอรี่ของร้านเอง)*
- **บริบท:** รับงานส่งของจากร้านไปหาลูกค้าในละแวก
- **เป้าหมาย:** เห็นรายการที่ต้องส่ง, ที่อยู่/แผนที่, กดยืนยันส่งสำเร็จ
- **ขับ requirement:** มุมมองไรเดอร์ (อาจเป็นเฟสหลัง) — *v1 อาจใช้คนของร้านจัดการผ่านฝั่งแอดมินก่อน*
